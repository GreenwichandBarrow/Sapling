## Notes

Webinar walkthrough on combining Clay (data enrichment platform) with Claude Code (terminal coding agent) for go-to-market workflows. Sales Captain team shared their stack and tactics.

### Topics covered
1. Stack architecture: Clay for AI column validation and data API enrichment, Claude Code for local file access, browser-based scraping, bot bypass, and unattended scoring/logic. n8n hooks them together. Sales Captain runs 400+ workflows.
2. Case study: "list that couldn't be bought" — UK cyber-essentials certified company database protected by Cloudflare. Used Claude Code to build a real-browser scraper that paste-typed 676 letter combinations into the search box, then used Clay + SERPR + OpenAI column to match company names to the right domain (no-confident-match returned nothing to prevent wrong-company outreach).
3. Cost economics: 100K personalized email lines for $7 in API + $200/mo Claude Code plan. Frame is cost-relative-to-value, not cheapest stack.
4. Workflow tips: plan in Claude AI chat first (cheaper than burning Claude Code tokens), use Sonnet sub-agents inside Claude Code for free ICP filtering, screenshot errors to Claude AI not Claude Code for debugging, ask Claude to fetch docs and paste back into Claude Code, always preview 3-5 rows before running at scale, clear context window at 50%.
5. CLAUDE.md file: 250-line hard ceiling per Bill, beyond that it hallucinates. After a successful workflow, ask Claude to convert the conversation into a reusable skill.

### Key takeaways
1. Clay and Claude Code are complementary, not substitutes. Clay owns enrichment and AI columns; Claude Code owns scraping bypass, local files, unattended logic.
2. Niche-database-as-source is Bill's "favorite trick to increase relevance" — find a database the prospect is in, write a static message that names it as the insight. Static beats AI messaging when the source itself is the personalization.
3. Supabase as a long-running prospect cache: lookup before re-enriching, refresh emails older than 3 months. Saves credits, compounds over time.
4. The Clay + Claude AI + Claude Code triangle: brief in Claude AI, paste prompt into Claude Code, output back to Clay for validation. Don't do everything inside the terminal.
5. Some teams have replaced Clay entirely with direct people-API + company-API access from inside Claude Code when use cases are simple enough.
