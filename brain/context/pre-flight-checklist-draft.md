# Pre-Flight Checklist — DRAFT

Run this scan before writing any sheet, doc, deck, or committing any deliverable. Goal: stop surface-level completion from masquerading as verified work. Pending Kay's review in next session.

## Accuracy & Depth

1. **Am I filling with surface-level information?** (Kay's add — names the "eagerness to complete" failure mode.) If the cell or line is shaped like a completed task but I haven't actually verified it against a source, STOP. Name it unverified, or run the verification.
2. **Is this a claim about the world?** If yes → verify against authoritative source (Gmail, Attio, HTTP fetch, tracker, Drive doc). Never from narrative text in SKILL.md, memory files, or prior conversation.
3. **Do I have verification-grade evidence for this cell/line?** If no → leave blank, mark "Needs verification," or spawn a subagent. Never fabricate or infer.

## Scope & Conventions

4. **Is this in scope for the skill I'm working on?** Deal-aggregator = prepped deals only, no lists. Target-discovery paused. Outreach-manager = drafts on existing lists.
5. **Am I building a maintained list/catalog/map?** If yes → check scope memories; most skills don't maintain lists.
6. **Did I use "Kay" anywhere?** Replace with "G&B" — any artifact outside private conversation.
7. **Did I reference columns by letter (Col A, Col B)?** Switch to field names.
8. **Am I inferring from narrative text in a spec/SKILL.md?** Narrative ≠ verification.

## Step Completion

9. **Is the prior step complete and verified?** Finish before pivoting. No deferred corrections.
10. **Would a session restart right now lose work?** If yes → finish, commit, or save state before the next topic.

## Decision Framing

11. **Am I about to recommend dropping a channel?** Never. Improve source quality within the channel.
12. **Am I about to auto-reject based on missing data?** Stop — Data Availability Rule. Absence flags for review, never rejects.
