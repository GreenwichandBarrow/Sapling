Approval gates are HARD STOPS. When Kay sets up a review/approve step (like customer validation docs before sending to JJ), that gate cannot be bypassed.

**Why:** Kay said "pronto" meaning "let's focus on reviewing and approving these quickly." I interpreted it as "send them now." This caused JJ to receive unreviewed call lists and required a hold message to walk it back.

**How to apply:**
- "Pronto", "ASAP", "get JJ on this" = prioritize the REVIEW, not skip it
- Present the docs for Kay's review immediately
- Only send to JJ after explicit "approved" or "send it"
- If unclear whether Kay is approving or asking for review, assume review
- The only way to bypass an approval gate is Kay explicitly saying "skip the review" or "just send it"
