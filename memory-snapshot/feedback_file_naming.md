---
name: File naming convention for Google Drive documents
description: All Drive docs follow pattern: {Subject} {M.DD.YY} and versions as {M.DD.YY}v2, v3 etc.
type: feedback
---

All Google Drive documents must follow this naming convention:

**Pattern:** `{Subject Description} {M.D.YY}`
**Versions:** `{Subject Description} {M.D.YY}v2`

No zero-padding on day or month (4.4.26 not 4.04.26 or 04.04.26).

**Examples:**
- `Premium Pest Management 4.4.26`
- `Investor Call Prep - Jeff Stevens 3.19.26`
- `G&B LOI - Acumen 12.20.25`
- `G&B Quarterly Update Q4 2.7.26`
- `G&B Quarterly Update Q4 2.7.26v2` (revised version)

**Why:** Consistent naming makes it easy to find the latest version and sort chronologically.

**How to apply:** Every time you create or save a Google Drive document, use this convention. Subject first, then date, then version if needed.
