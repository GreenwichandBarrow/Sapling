Customer validation call lists for JJ follow this exact process:

1. Claude preps the call list docs (contacts, script, diagnostic questions)
2. Kay reviews and approves each doc individually
3. For each approved doc, Claude drafts the Slack message and presents to Kay for approval
4. Once Kay approves the message, Claude sends ONE Slack message per niche to JJ
5. Each message includes the call list link AND a link to the niche one-pager for background

**Why:** JJ needs context on what the niche is to have informed calls. One message per niche keeps things clear and lets Kay approve the rollout pace. Kay reviews everything before it reaches JJ.

**How to apply:**
- Never batch all niches into one message
- Always include the one-pager link in the Slack message
- Always show Kay the draft message before sending
- Kay approves both the doc content AND the Slack message separately
- Add feedback request at the bottom: "Any feedback on this process at all along the way is welcome and appreciated"
