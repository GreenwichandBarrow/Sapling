Never ask "should I add this to WEEKLY REVIEW?" for a niche that hasn't been through the pipeline. That puts Kay and her analyst in front of a niche with no one-pager, no score, no target pool data — nothing to discuss.

**The correct question:** "Should I run these through niche-intel starting at Step 2 (Identify)?"

**The flow:** Identify → One-Pager → Score → Add to tracker. By the time a niche reaches WEEKLY REVIEW, it has a one-pager, industry scorecard score, target pool estimate, and named companies.

**Why:** Kay pointed out that randomly adding niches to WEEKLY REVIEW puts her analyst in a bad position — they need the deliverables to have a productive discussion. The pipeline exists for a reason.

**How to apply:** When new niche ideas surface (from calls, emails, contacts, research), the system queues them for the next niche-intel run or offers to run them through the pipeline immediately. They never skip steps.
