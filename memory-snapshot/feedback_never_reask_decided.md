Never ask Kay a question she's already answered. If the system needs confirmation, present her PRIOR answer back to her ("Last time you said X — still correct?") instead of asking the question fresh.

**Why:** Kay doesn't want to repeat herself. Re-asking wastes her time and signals the system isn't learning. Decisions made in sessions should be durable.

**How to apply:**
- Before asking any question, check session notes, memory, and vault for prior decisions on the same topic
- If a prior answer exists, state it: "Per your decision on [date], [answer]. Proceeding accordingly." or "You previously said X — confirming?"
- If no prior answer exists, then ask — but only once
- This applies to pipeline decisions, niche calls, outreach approvals, and all other recurring questions
- Pairs with feedback_close_the_loop: decisions should be actioned AND remembered
