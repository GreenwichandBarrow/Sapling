Every skill and workflow should trend toward removing Kay from manual steps. We are the office of continual improvement.

**Why:** Kay's time is the scarcest resource. Every manual review step she does is time not spent on relationship-building and deal-making — the things only she can do. The system should handle everything else.

**How to apply:**
- When building or updating any skill, ask: "Which steps currently require Kay that could be automated?"
- Use the Skill Calibration tab (Weekly Tracker) to track phase testing → graduation for every human-in-the-loop step
- Pattern: Active Review → Spot Check → Auto-Advance. Kay decides when to graduate, but the system proposes it with data.
- Don't just remove steps — prove they're reliable first (accept rate, error rate, consecutive clean weeks)
- This applies to ALL skills, not just target-discovery: outreach-manager, niche-intelligence, pipeline-manager, deal-evaluation
- When designing new skills, build the graduation path in from day one — don't bolt it on later
