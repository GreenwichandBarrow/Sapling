---
name: Always persist pending items before conversation ends
description: Save memories and/or inbox items for anything the user is waiting on or needs to follow up on
type: feedback
---

When a conversation involves a pending action (waiting for an email, export, response, etc.), always persist it before the conversation ends — either as a project memory or a brain/inbox item.

**Why:** Kay had to re-explain a pending ChatGPT data export because it wasn't saved from the prior conversation. Context was lost.

**How to apply:** At the end of any conversation where there are open loops — things the user is waiting on, actions not yet completed, follow-ups needed — save a project memory and/or create an inbox item so the next conversation has full context.
