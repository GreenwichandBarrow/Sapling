---
name: Tabled vs Ideation distinction
description: TABLED = thoroughly reviewed, parked with trigger condition. IDEATION = not fully vetted yet. Never confuse them.
type: feedback
---

TABLED = niches that have been thoroughly reviewed (one-pager, scorecard, full evaluation) and parked because something specific would need to change for them to work. They have a trigger condition.

IDEATION = niches that haven't been fully reviewed yet. Ideas, signals, early-stage candidates that still need one-pagers and scorecards.

**Why:** Kay corrected this when 19 Under Review niches were incorrectly moved to TABLED instead of IDEATION. Under Review niches haven't been fully vetted, so they belong in IDEATION.

**How to apply:** When moving niches off WEEKLY REVIEW, ask: "Has this been fully evaluated (one-pager + scorecard)?" If yes → TABLED. If no → IDEATION.
