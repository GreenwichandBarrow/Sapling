# Buyer-Fingerprint / Searcher-Fit First (2026-04-28)

**Rule:** Sourcing decisions, niche scoring, target outreach, and industry-thesis drafting must filter THROUGH a buyer fingerprint that captures who Kay is built to run — not just what financial criteria she'll accept. Identity precedes industry.

**Why:** Three converging signals.
1. Kay's investors (Guillermo, Jeff, others) believe strongly in searcher-fit + going deep on a specific operator profile rather than casting wide. This is investor doctrine, not preference.
2. Industry-thesis drafting has been a chronic pain point. The niche pipeline keeps cycling through new ideas without a stable upstream filter, leading to surface-level scoring and abandoned threads. The missing layer is identity-first.
3. Athena Simpson's MatchMaker pitch (2026-04-28) named this directly — "fingerprint not filter," 6-dimension buyer-fit profile drives match, financial criteria are derivative.

**How to apply:**
- Before any new niche enters scoring, check whether it fits the buyer fingerprint — not just the financial buy-box. If a niche scores well financially but fails fingerprint, surface the conflict; don't auto-advance.
- When deal-aggregator surfaces a deal, the match narrative should reference fingerprint dimensions ("matched on operator-profile + lifestyle-fit; concern: risk-tolerance disclosure"), not just thesis match.
- When Kay says "what about industry X?", probe fingerprint fit before launching market research. Save research effort on poor-fit ideas.
- Fingerprint lives in `brain/context/buyer-fingerprint.md` (build started 2026-04-28).
- Re-run / refresh the fingerprint as Kay gets reps. Geography tightens, financial ranges shift, criteria recalibrate. The fingerprint is a living doc, not a one-time exercise (per Athena's "Lady Ebetah re-runs" model).

**Don't apply this rule to:**
- Stress-test hypotheticals where Kay is deliberately inverting filters to pressure-test analysis (per `feedback_stress_test_constraints_not_facts`).
- Active-deal evaluation past the LOI stage — by that point, fit has been validated; don't re-litigate.
