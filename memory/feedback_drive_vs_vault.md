---
name: Google Drive vs vault file strategy
description: CRITICAL - Drive has living docs (always check there first), vault has one-and-done snapshots
type: feedback
---

Google Drive = living, evolving documents. One-pagers, scorecards, target lists, trackers, call logs. These change over time. ALWAYS check Drive for the latest version, never trust a vault copy of a dynamic file.

Vault (brain/) = one-and-done snapshots. Call notes, entity files, decision traces, daily notes. Created once, referenced later.

**Process:** Files are created in Google Drive first. Once finalized (one-and-done), copy into brain/ as a reference snapshot. Dynamic files stay in Drive only.

**Why:** Kay found the system reading a stale/empty vault version of the art advisory one-pager when the real, robust version was in Drive. The vault copy was outdated.

**How to apply:** When reviewing any deliverable (one-pager, scorecard, target list), ALWAYS pull from Google Drive first. Only fall back to vault if Drive is unavailable. When creating new deliverables, create in Drive first, then snapshot to vault if it's a one-and-done.
