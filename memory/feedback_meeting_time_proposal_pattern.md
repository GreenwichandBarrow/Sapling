**The Harrison template (verbatim, for reference):**
> Hey Kay,
>
> Great to hear from you. I'd definitely be up for a call.
>
> Before we get into the specific issues, I want to use part of the conversation to talk about what the right setup looks like going forward. [optional forward-framing sentence]
>
> I have a couple windows this week if either works. If not, @Howie can help us find some time.
>
> Tomorrow 4/29 at 2:00 PM CST
> Thursday 4/30 at 12:00 PM CST
>
> Best,
> Harrison

**The reusable structure:**
1. Warm opener acknowledging the reach-out ("Great to hear from you" / "Good to connect again")
2. Optional one-sentence forward framing — what Kay wants to use the call for, or what she wants to make sure they cover. Skip if not relevant.
3. Lead-in line offering windows: "I have a couple windows this week if either works. If not, [fallback] can help us find time."
4. Exactly **TWO** specific options. Format: `Day Date at Time TZ` — e.g. "Tuesday 5/6 at 10:00 AM ET". Day name + month/day + time + timezone, all four. Two options is the discipline — not three, not "let me know."
5. Sign-off: standard Kay close ("Very best, Kay" — signature is built in via Superhuman).

**Why this works:**
- Two windows is enough for the recipient to commit but not so many that they need to pick from a list
- Looping in a scheduling fallback (Howie analog for Harrison) prevents the email tennis if neither lands
- Forward framing sets the expectation for the meeting before the meeting — recipient comes prepared
- Day + date + TZ together kills timezone ambiguity

**How to apply:**
- Use whenever Kay is the one **offering** her availability (replying to "let's grab a call" / responding to a warm intro / following up on a conference contact who said "we should chat").
- Do **NOT** use when Kay is **picking** from windows the other side already sent (e.g., Andrew Lowis via Barrie — that's just "Wed May 6 10am works, talk then"). Different shape.
- Pick the fallback-loop-in based on who's helping Kay schedule. Default: Calendly link if Kay has one, else "happy to send more options" or a specific helper. If no scheduling tool exists yet, this is a future Kay-needs-this-set-up gap to flag.
- Two windows max. Three is a list to scan; two is a commit-or-counter.
- Day-aware sign-off still applies per `feedback_day_aware_signoffs.md` — match the SEND day's tone.
