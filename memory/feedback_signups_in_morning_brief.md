Whenever the deal-aggregator, outreach-manager, conference-discovery, or any sourcing skill has a source blocked on Kay's registration (platform signup, newsletter subscription, association membership, email-alert toggle), surface them as a dedicated line item in the morning briefing — with the direct signup/registration URL next to each. Don't bury the unblock path inside a skill artifact or a "this week" task.

**Format:**
- Group by priority (platforms → email alerts → specialists → associations)
- One row per source with: source name, what it adds (which niche / what coverage), signup link, action verb
- Flag dollar cost where membership fees apply (associations especially) — CFO verdict needed before committing
- Carry the list forward in the morning brief until each signup is done or formally declined

**Why:** 2026-04-21 — We had agreed on 4/20 that the 8am deal-aggregator run would include all newly-added sources, AND that the morning brief would surface the list of sources needing Kay's signup with links. I delivered neither — the skill file had the sources but the scan path didn't execute them, and I dropped the signups line item from the briefing entirely. Kay caught both misses in the same morning. The signup list is the critical unblock path; if it isn't surfaced with links, nothing ships.

**How to apply:** Every morning brief assembly — before rendering — grep every active skill's source list for `register`, `membership`, `subscription`, `sign up`, `buyer registration`, `login-gated`, `requires` and collect into a "Signups / Registrations Pending" section. Include the signup URL for every entry. Pending signups belong in Dropped Balls if they've aged >7 days, otherwise This Week. Never hide them in a skill artifact that Kay has to go find.
