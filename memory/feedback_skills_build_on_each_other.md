Skills must build on each other like a crescendo. Don't duplicate work across skills.

Flow: niche-intelligence → target-discovery-outreach → pipeline-manager
- niche-intelligence: produces one-pager, scorecard, buy-box validation, ICP, Linkt ICP
- target-discovery-outreach: consumes those outputs, finds targets, drafts outreach
- pipeline-manager: takes targets from discovery, manages stage progression, nurture, follow-ups
- conference-prep: feeds into pipeline-manager and target-discovery with attendee lists

**Why:** Kay flagged (2026-03-18) that target-discovery-outreach Phase 1 was duplicating niche-intelligence work (writing thesis, defining ICP) and Phase 4 was duplicating pipeline-manager (stage progression, tracking). Each skill should only do what's unique to it.

**How to apply:** When building a new skill, check what already exists upstream. Reference it, don't rebuild it. Inputs come from prior skills, outputs go to downstream skills.
