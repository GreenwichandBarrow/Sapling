---
name: Build template first, then execute from it
description: Always build/update the template SKILL before taking action — don't act then reverse-engineer the skill
type: feedback
---

When building a new process (like JJ's call logs), build the TEMPLATE and SKILL first, then execute from it.

**Why:** On April 7, we built call log tabs with an ad-hoc 9-column format, then had to redo everything when Kay pointed out the columns should match the full target list. Building the template first would have prevented the rework. Kay's words: "This is why we always have to focus on building SKILLS and not just taking an action."

**How to apply:** For any new repeatable process: (1) discuss and agree on the format with Kay, (2) build/update the template, (3) update the skill to reference the template, (4) THEN execute. Never skip to execution.
