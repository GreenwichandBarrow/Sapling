## Rule

When Kay is iterating on an existing doc/template that already lives somewhere canonical (Drive, vault, etc.) and she's giving directional feedback ("this paragraph is wrong," "remove this section," "the tone is off"), push the edits directly to the live doc, then summarize what changed in chat. Do NOT show a preview draft and ask for explicit approval before pushing.

## Why

- **Iteration mode is different from bulk-draft mode.** `feedback_template_approval_before_bulk_drafts.md` says: for any skill drafting N>3 personalized outputs, show Kay the rendered template(s) FIRST. That's the *creation* path — preventing N bad drafts by reviewing the source. **Iteration is the opposite path** — Kay's already approved the source, now she's polishing it. Asking for approval on each polish edit is friction.
- Kay's pace during template iteration is fast and verbal-direction-driven. She says "remove the what we do section" and expects to see it gone, not a preview asking "should I remove this?"
- Live-doc edits are fully reversible (Drive history, gog docs edit). Risk is low; speed matters more.
- Calibrated 2026-04-30 after multiple turns of "show preview → Kay says yes → push edit" friction during the G&B Intermediary Outreach Templates doc revision.

## How to apply

**Preview-first mode (keep doing this):**
- Bulk drafts (N personalized emails / records) where Kay hasn't seen the template
- New skills producing first-time output
- High-stakes external sends where the wrong words hit a real recipient
- Any case where the destination is irreversible (sent email, public post, committed Git push)

**Live-edit mode (use this when iterating):**
- Kay says "update the doc as I give you feedback" or similar
- Kay is giving directional feedback on an existing doc/template/memory she built or approved
- Doc is in a canonical Drive location with version history (reversible)
- The change is structural/wording-level, not semantic-direction-level

**Always:** after pushing, summarize what changed in chat (3-5 bullets) so Kay can sanity-check without re-reading the whole doc. If multiple edits, batch the summary at the end of the sequence, not after each edit.

**If unsure which mode:** default to live-edit when Kay has explicitly told me to iterate fast (like the 2026-04-30 instruction), default to preview when in doubt and the cost of being wrong is high.
